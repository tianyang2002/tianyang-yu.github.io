Leetcode 209. Minimum Size Subarray Sum（长度最小的子数组）#

题目#

给定一个含有 n ****个正整数的数组和一个正整数 target 。

找出该数组中满足其和 ****≥ target **的长度最小的 连续子数组 [numsl, numsl+1, ..., numsr-1, numsr] ，并返回其长度。**如果不存在符合条件的子数组，返回 0

思路#

目标：用一层循环实现

滑动窗口法#

  1. 窗口：双指针内所有的元素（数字）
  2. 寻找子数组：计算窗口内的元素和判断是否与target相等
  3. 长度最小子数组的选择：res = length < res ? length : res

如何移动子数组#

  1. 将双指针的后指针往后移，并计算窗口内元素的和，如果窗口内的元素和大于或者等于target，则计算当时窗口的长度，并判断是否是最小的子数组，然后将双指针的前指针向后移动，直到窗口内元素的和小于target为止。不停的重复直到后指针到达了数组最后一个元素
		int sum = 0;
        int i = 0;
        int res = INT32_MAX;
        for (int j = 0; j < nums.size(); j++) {
            sum += nums[j];
            while (sum >= target) {
                int length = j - i + 1;
                res = length < res ? length : res;
                sum -= nums[i];
                i++;
            }
        }
        return res == INT32_MAX ? 0 : res;

问题#

  1. 为什么这样移动能包含所有的和等于target的子数组？

因为在窗口移动时，只有当窗口内元素和≥target时才能代表窗口内可能包含满足和等于target的子数组，可能在后指针移动时已经发现（sum == target），可能在前指针往后移动时（sum > target），直到（sum == target）发现，可能发现不了，就继续以这种方法往后探

代码#

class Solution {
public:
    int minSubArrayLen(int target, vector<int>& nums) {
        int sum = 0;
        int i = 0;
        int res = INT32_MAX;
        for (int j = 0; j < nums.size(); j++) {
            sum += nums[j];
            while (sum >= target) {
                int length = j - i + 1;
                res = length < res ? length : res;
                sum -= nums[i];
                i++;
            }
        }
        return res == INT32_MAX ? 0 : res;
    }
};

复杂度分析#

  • 时间复杂度：O(n)
  • 空间复杂度：O(1)

时间复杂度为什么是O(n)？

外层的for-loop是O(n)复杂度，但是内层的while循环并不是O(n)复杂度（i可移动的距离是j-i），而是一个常熟级复杂度O(1)，所以总的复杂度是O(n)

如果是[0, 0, 0, …….., 200], ………部分都是0, target是200, 那么while的复杂度就是O(n)了. 可是由于不是每次for-loop都有一个O(n)的while, 所以复杂度还是O(n)

Leetcode 59. 螺旋矩阵 II#

题目#

给你一个正整数 n ，生成一个包含 1 到 n2 所有元素，且元素按顺时针顺序螺旋排列的 n x n 正方形矩阵 matrix

思路#

由于螺旋旋转中数字的填充是每一次加1, 将问题规模缩小, 并且能够解决此规模的问题就可以解决原规模的问题. 所以将问题缩小成一圈. 对于输入数据n, 生成旋转矩阵需要n/2圈, 当n为偶数, n/2可行, 但是当n为奇数时, n/2不可行, 会多一个数据空间, 对于处理这个数据空间, 只需要把最后的那个数据赋值进去就可以了.

if (n % 2 == 1) {
	res[n / 2][n / 2] = count;
}

对于处理每一圈, 将用四个for循环来处理每个问题的边界及从那个位置到那个位置结束.

    int x = 0;
		int y = 0;
		int count = 1;

    int offset = 0;
		int i = x;
    int j = y;
    for (j = y; j < n - offset; j++) {
        res[x][j] = count++;
    }
    for (i = x; i < n - offset; i++) {
        res[i][j] = count++;
    }
    for (; j > y; j--) {
        res[i][j] = count++;
    }
    for (; i > x; i--) {
        res[i][j] = count++;
    }
    x++;
    y++;
    offset++;

其中:

  • offset: 代表每多一圈, 后边界就要缩一格
  • x, y代表数组的列和行, x, y的值代表每一圈开始转的位置
  • 4个for循环代表4个边上赋值的操作

最后利用一个while循环来执行所需要的圈数(n/2)

class Solution {
public:
    vector<vector<int>> generateMatrix(int n) {
        int x = 0;
        int y = 0;
        int offset = 1;
        int count = 1;
        int loop  = n / 2;
        vector<vector<int>> res(n, vector<int>(n));
        while (loop--) {
            int i = x;
            int j = y;
            for (j = y; j < n - offset; j++) {
                res[x][j] = count++;
            }
            for (i = x; i < n - offset; i++) {
                res[i][j] = count++;
            }
            for (; j > y; j--) {
                res[i][j] = count++;
            }
            for (; i > x; i--) {
                res[i][j] = count++;
            }
            x++;
            y++;
            offset++;
        }
        if (n % 2 == 1) {
            res[n / 2][n / 2] = count;
        }
        return res;
    }
};

复杂度分析#

  • 时间复杂度：O(n^2)

while循环的复杂度为O(n), while循环里面的复杂度是由4个for循环决定的, 为O(n), 所以总时间复杂度为O(n^2)

  • 空间复杂度：O(1)
